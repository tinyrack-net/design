# Radius

Shape tokens keep controls crisp and reserve roundness for explicit roles.

## Principle

Radius should make controls recognizable without softening the whole interface. Choose the shape by role, then keep nested elements equal to or tighter than their containing surface.

| Role | Range | Rule |
| --- | --- | --- |
| Control | `sm - md` | Interactive elements stay compact and predictable. |
| Surface | `lg - xl` | Containers can be softer only when they hold a distinct layer. |
| Pill | `full` | Use only when the shape communicates badge, status, or circle. |

## Token reference

| Token | Value | Use |
| --- | --- | --- |
| `xs` | `0.125rem` | Keyboard hints, tiny badges, checkboxes, icon containers. |
| `sm` | `0.25rem` | Tags, dense inputs, compact controls, table imagery. |
| `md` | `0.375rem` | Buttons, inputs, segmented controls, navigation items. |
| `lg` | `0.5rem` | Panels, popovers, dropdowns, distinct operational surfaces. |
| `xl` | `0.75rem` | Modals and larger containers above the page rhythm. |
| `full` | `9999px` | Status capsules, avatars, circular indicators only. |
