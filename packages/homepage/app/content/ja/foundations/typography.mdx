## 階層を選ぶ

読む順序を先に決め、その順序を表現できる最小のロールを選んでください。`display` はまれなページレベルの文、見出しロールは入れ子のセクション、`body` と `bodySm` は本文、`caption` は補足、`label` は短い識別子に使います。

## 文字ロール

'ラック状態' : name === 'code' ? 'rack.deploy()' : 'クラスターの状態と容量'}</span><TRCode>{name}</TRCode></div>))}</div>

## 行長とローカライズ

段落は読み取りやすい幅に抑え、ラベルは伸びられるようにしてください。日本語や韓国語は英語と異なる位置で改行されます。英語の文字列を基準に幅を固定したり、説明文を 1 行に強制したりしないでください。大文字の字間は短いラテン文字のラベルだけに使い、日本語の本文には適用しません。

## フォント読み込みの責任

Core は `body`、`heading`、`mono` トークンに `"IBM Plex Sans"` というファミリー名だけを宣言します。フォントファイルの提供や読み込みは行いません。利用するアプリが必要なウェイトと言語サブセットを読み込んでください。このドキュメントアプリは IBM Plex Sans と韓国語・日本語のファミリーを読み込みます。

## 実装とリファレンス

```css
@import "@tinyrack/ui/core.css";
.rack-title { font: var(--tinyrack-weight-heading) var(--tinyrack-text-2xl) / var(--tinyrack-leading-sm) var(--tinyrack-font-heading); }
```
