## セマンティックロールを選ぶ

色の値ではなく、意味から決めてください。

- **表面:** canvas、surface、muted surface、hover、selected、pressed はレイヤーと操作を表します。
- **内容:** text、muted text、placeholder、border、inverse は前景色を文脈に結び付けます。
- **操作:** primary、hover、pressed、on-primary は 1 つの操作セットです。
- **ステータス:** info、success、warning、danger には、内容、表面、操作、境界線、on-color の各ロールがあります。

## テーマ構造

祖先要素に `data-theme="tinyrack-light"` または `data-theme="tinyrack-dark"` を設定してください。両方のセレクターが同じセマンティック変数を定義するため、テーマが変わっても子要素の意図は維持されます。

```html
<html data-theme="tinyrack-light">
```

通常、hover、pressed のロールをセットで使います。light テーマから dark の値を算出したり、無関係な塗りに前景色を載せたりしないでください。

<div className="flex flex-wrap gap-2"><TRButton intent="primary">デプロイ</TRButton><TRButton intent="info">詳細</TRButton><TRButton intent="success">承認</TRButton><TRButton intent="warning">リスク確認</TRButton><TRButton intent="danger">削除</TRButton></div>

## Primary を安全に拡張する

Tinyrack のテーマセレクターを維持し、別の製品用アクセントセレクターを追加します。通常、hover、pressed、前景のコントラストを 1 つの契約として保つため、4 つのロールをすべて上書きしてください。

```css
[data-accent="rack-blue"] {
  --tinyrack-primary: #1d4ed8;
  --tinyrack-primary-hover: #1e40af;
  --tinyrack-primary-pressed: #1e3a8a;
  --tinyrack-on-primary: #ffffff;
}
```

```html
<main data-theme="tinyrack-dark" data-accent="rack-blue">
```

## 避ける使い方

- プロダクト UI でパレット値を直接使わないでください。テーマに追従しません。
- ステータス色を装飾だけに使ったり、warning と danger を入れ替えたりしないでください。
- `primary` だけを上書きしないでください。状態セットが不完全だと操作フィードバックが一貫しません。
- `on-*` 色を別の背景ロール上に置かないでください。

## 実装

```css
@import "@tinyrack/ui/core.css";
.rack-status { color: var(--tinyrack-info); background: var(--tinyrack-info-surface-subtle); border-color: var(--tinyrack-info-border); }
```

接頭辞付きユーティリティは [Tailwind の色マッピング](/ja/foundations/tailwind/#colors)で確認してください。
