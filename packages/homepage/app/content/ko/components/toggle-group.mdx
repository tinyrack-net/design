## 계약

| 항목 | 계약 |
| --- | --- |
| 선택 | 기본값은 0개 또는 1개이며, `multiple`로 여러 값을 독립적으로 선택 |
| 상태 | 제어형 `value` 또는 비제어형 `defaultValue`; 둘 다 문자열 배열을 사용하고 기본값은 `[]` |
| 방향 | 기본값은 가로이며 필요할 때 세로로 설정 |
| 키보드 | 방향에 맞는 화살표와 Home/End 사용; 기본적으로 순환하며 `loopFocus={false}`이면 끝에서 정지 |
| 사용 가능 여부 | 기본적으로 활성화되며 group 전체 또는 개별 item을 비활성화 가능 |

## API

### 구조

```tsx
<TRToggleGroup aria-label="텍스트 정렬">
  <TRToggle value="start">시작</TRToggle>
  <TRToggle value="center">가운데</TRToggle>
</TRToggleGroup>
```

`TRToggleGroup`은 상태와 roving focus를 관리하는 root예요. 자식은 `@tinyrack/ui/components/toggle`에서 가져온 공개 `TRToggle` 컴포넌트이며, group 안의 각 toggle에는 안정적이고 고유한 `value`가 필요해요.

### Props

| Prop | Type | 기본값 | 용도 |
| --- | --- | --- | --- |
| `value` | `readonly string[]` | - | 제어형 pressed 값이에요. |
| `defaultValue` | `readonly string[]` | `[]` | 비제어형 pressed 값의 초기값이에요. |
| `onValueChange` | `(value, details) => void` | - | 다음 배열과 native event 세부 정보를 전달해요. `details.cancel()`로 Base UI의 내부 업데이트를 취소할 수 있어요. |
| `multiple` | `boolean` | `false` | 각 item을 독립적으로 toggle할 수 있게 해요. |
| `orientation` | `'horizontal' \| 'vertical'` | `'horizontal'` | layout과 활성 arrow-key 축을 선택해요. |
| `loopFocus` | `boolean` | `true` | 첫 번째와 마지막 활성 item 사이에서 arrow-key focus를 순환해요. |
| `disabled` | `boolean` | `false` | group의 모든 item을 비활성화해요. |
| `render` | `ReactElement \| function` | `<div>` | 동작과 병합된 ref/props를 유지하면서 root를 교체해요. |

Root는 `role="group"`으로 렌더링되므로 `aria-label` 또는 `aria-labelledby`를 제공하세요. 가로에서는 Left/Right, 세로에서는 Up/Down으로 focus를 이동하고 Home/End는 첫 번째/마지막 활성 item으로 이동해요. 비활성화된 item은 roving focus에서 건너뛰어요. Group `disabled`는 모든 item에 적용되지만 개별 `TRToggle`은 독립적으로 비활성화할 수 있어요.

`className`, `style`, native `div` props, event handler, ref가 전달돼요. `--tr-toggle-group-gap`과 `--tr-toggle-group-color`로 간격과 텍스트 색상을 바꾸고, toggle 자체의 appearance는 `TRToggle` token으로 조정하세요. Toggle group은 이름 있는 form 값을 제출하지 않으므로 selection이 form 제출에 참여해야 한다면 `TRCheckboxGroup` 또는 `TRRadioGroup`을 사용해요.
