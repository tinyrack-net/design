## 계약

| 축 | 값 | 기본값 |
| --- | --- | --- |
| state | controlled `open` or uncontrolled `defaultOpen` | closed |
| availability | `disabled` | interactive |
| lifecycle | unmount, `keepMounted`, or `hiddenUntilFound` | unmount |

## API

### `TRCollapsible.Root`

| Prop | 타입 / 기본값 | 용도 |
| --- | --- | --- |
| `open` / `defaultOpen` | `boolean` / `undefined`, `false` | controlled expansion 또는 uncontrolled 초기 상태 |
| `onOpenChange` | `(open, details) => void` | trigger와 browser find에 의한 변경을 알림 |
| `disabled` | `boolean` / `false` | trigger를 focus로 찾을 수 있게 유지하면서 상태 변경을 차단 |
| `render`, `className`, `style`, `ref` | Base UI root props | 동작을 유지하며 `<div>` host를 맞춤 설정 |

### `TRCollapsible.Trigger`

`aria-expanded`와 `aria-controls`를 가진 button을 render해요. native button props, `render`, style, ref를 받아요. Enter와 Space로 토글해도 focus는 trigger에 남고, disabled trigger는 `data-disabled`와 `aria-disabled`를 노출해요.

### `TRCollapsible.Panel`

닫힌 content를 DOM에 유지하려면 `keepMounted`, browser page search가 표시할 수 있게 하려면 `hiddenUntilFound`를 사용해요. 둘 다 지정하면 `hiddenUntilFound`가 우선해요. 기본 lifecycle은 closing transition 뒤 unmount해요. panel은 `--collapsible-panel-height`, `--collapsible-panel-width`, `data-starting-style`, `data-ending-style`을 노출해요.

닫힌 SSR output은 선택한 lifecycle을 따르며 public relationship을 바꾸지 않고 hydrate돼요. Tinyrack CSS는 design-system motion token을 사용하고 reduced motion에서는 장식 transition을 제거해요. `TRCollapsibleRoot`, `TRCollapsibleTrigger`, `TRCollapsiblePanel`, 각 prop type과 Base UI state type은 `@tinyrack/ui/components/collapsible`에서 export되며 CSS는 opt-in이에요.
