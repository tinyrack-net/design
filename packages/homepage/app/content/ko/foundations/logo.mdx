## 배치

각 surface에 lockup을 한 번만 사용하고 composition의 안정적인 가장자리에 배치하세요. left-to-right interface에서는 top-left를 선호하며 launch, loading, end-frame 맥락에는 중앙 배치가 적합해요. favicon과 service avatar 같은 반복되는 컴팩트 surface에는 mark를 사용하세요.

제품, 프로젝트, partner 이름은 lockup clear space 밖에 두세요. 새 logo를 확장하거나 조판하지 말고 **Tinyrack UI**를 일반 interface text로 작성해요.

## 접근성

logo가 link의 유일한 콘텐츠라면 <TRCode>alt=&quot;Tinyrack&quot;</TRCode>을 사용해 link에 간결한 접근 가능한 이름을 제공하세요. 주변 text가 이미 Tinyrack을 이름 짓고 image가 중복되면 빈 alt attribute를 사용해요.

```html
<a href="/">
  <img src="/brand/tinyrack-lockup.svg" alt="Tinyrack" width="156" height="38" />
</a>
```

rack 그림을 설명하려고 alt text에 추가 문구를 넣지 마세요. 접근 가능한 이름은 brand를 식별하고 page copy가 artwork를 설명해요.

## 형태와 구성

마크는 정렬된 상태 구멍이 있는 세 단의 랙 베이로 이루어져요. 로고 조합에서는 외곽선으로 변환한 IBM Plex Sans SemiBold 워드마크와 마크, 고정 간격이 하나의 원본을 이뤄요. 각 부분을 다시 입력하거나 분리하거나 재배치하지 마세요.
