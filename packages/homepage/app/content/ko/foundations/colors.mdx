## 시맨틱 역할 선택

색상 값보다 의미부터 정하세요.

- **표면:** canvas, surface, muted surface, hover, selected, pressed는 레이어와 상호작용을 나타내요.
- **콘텐츠:** text, muted text, placeholder, border, inverse 역할은 전경색을 맥락에 연결해요.
- **동작:** primary, hover, pressed, on-primary는 하나의 동작 세트예요.
- **상태:** info, success, warning, danger에는 각각 콘텐츠, 표면, 상호작용, 테두리, on-color 역할이 있어요.

## 테마 구조

상위 요소에 `data-theme="tinyrack-light"` 또는 `data-theme="tinyrack-dark"`를 설정하세요. 두 selector는 같은 시맨틱 변수를 정의하므로 테마가 바뀌어도 하위 요소의 의도가 유지돼요.

```html
<html data-theme="tinyrack-light">
```

기본, hover, pressed 역할을 한 세트로 사용하세요. light 테마 값으로 dark 값을 계산하거나 관련 없는 채움에 전경색을 사용하지 마세요.

## Primary 안전하게 확장

Tinyrack 테마 selector는 유지하고 별도의 제품 accent selector를 추가하세요. 기본, hover, pressed, 전경 대비가 하나의 계약으로 유지되도록 네 역할을 모두 재정의하세요.

```css
[data-accent="rack-blue"] {
  --tinyrack-primary: #1d4ed8;
  --tinyrack-primary-hover: #1e40af;
  --tinyrack-primary-pressed: #1e3a8a;
  --tinyrack-on-primary: #ffffff;
}
```

```html
<main data-theme="tinyrack-dark" data-accent="rack-blue">
```

## 피해야 할 사용

- 제품 UI에서 팔레트 값을 직접 사용하지 마세요. 테마에 대응하지 못해요.
- 상태 색상을 장식에만 쓰거나 warning과 danger를 바꾸어 쓰지 마세요.
- `primary` 하나만 재정의하지 마세요. 상태 세트가 불완전하면 상호작용 피드백이 일관되지 않아요.
- `on-*` 색상을 다른 배경 역할 위에 놓지 마세요.
